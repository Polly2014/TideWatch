# TideWatch Digest Skill 需求文档

Jerry，这是你的第一个 skill！由小龙虾 🦞 设计，保利拍板，交给你来实现。

## 背景

你和 TideWatch 住在同一台 Azure VM 上，可以本地直接调用 MCP 接口（`localhost:8889`）。你已经能拉到原始分析数据了，但现在发到微信的内容太"报表"——像 Excel 导出，不像朋友聊天。

**目标：把 TideWatch 的数据变成一份微信友好的每日投研简报。**

## 核心原则

微信是聊天场景，不是仪表盘。报告要像一个懂行的朋友下班后跟你聊今天的盘面。

### 文字规范

1. **开头一句话定调** — 今天大盘什么心情，一句话概括（"今天大 A 集体摸鱼"、"港股领涨，A 股跟了个寂寞"）
2. **持仓只说有变化/需要关注的** — 没变化的不逐条展开，一笔带过（"其他几只继续躺着"）
3. **每只股票先说结论再说原因** — 别先列一堆指标再推结论，反过来："继续等——MACD 刚转红但资金还在流出"
4. **有态度** — 别停在"中性观望"，给个方向感（"继续等"/"可以考虑补"/"该想想了"）
5. **控制长度** — 微信消息太长没人看，文字部分控制在 **300 字以内**
6. **emoji 克制** — 全文 2-3 个就够，别每行一个

### 图表规范

这是画龙点睛的部分！每日简报配 **1-2 张图表**，让数据一目了然。

**必选图（每日一张）：**
- **持仓概览图** — 横向柱状图或卡片式布局
  - 每只股票一行：名称 + 今日涨跌% + 持仓浮盈/亏%
  - 红涨绿跌（A 股配色）
  - 底部一行：总持仓盈亏
  - 数据来源：`scan_market` → `holdings[]` → 每只自带 `pnl_pct`、`pnl_amount`、`pct_today`，直接汇总即可，**不需要**单独调 `manage_holdings` 再自己算

**可选图（有信号/异动时才出）：**
- **个股 K 线缩略图** — 最近 20 日 K 线 + MA5/MA20 + 成交量
  - 只在有明确信号时出（比如突破/跌破关键均线、放量异动）
  - 不需要每只都画
- **资金流向图** — 简单的柱状图，主力/散户净流入流出
  - 只在大额异动时出

**图表技术方案：**
- 用 Python `matplotlib` 生成 PNG，然后通过微信发图片
- 图片风格：白底、简洁、大字号（手机上要看得清）、A 股红涨绿跌
- 尺寸：宽 800px，高度按内容自适应
- 建议在 VM 上写个 Python 脚本专门负责画图，skill 里调用
- 输出路径：`/tmp/tidewatch_digest_{YYYYMMDD}.png`（用 /tmp 避免磁盘堆积，带日期防冲突）
- **VM 无头环境必需配置：**
  ```python
  # tidewatch_chart.py 开头
  import matplotlib
  matplotlib.use('Agg')  # 无 GUI 环境必须用 Agg 后端
  import matplotlib.pyplot as plt
  plt.rcParams['font.sans-serif'] = ['Noto Sans CJK SC']
  plt.rcParams['axes.unicode_minus'] = False
  ```
  中文字体一次性安装：`sudo apt install fonts-noto-cjk`

### 消息结构

一次发送 **2 条微信消息**，顺序：

```
📊 消息 1：图片（持仓概览图）

📝 消息 2：文字简报
---
4/7 盘后

大盘：沪指 -1.33%，创业板 -0.8%，缩量阴线，短期别追高。

持仓：
· 和而泰 29.24（-2.53%）— 28-30 磨底中，MACD 转红但资金流出，继续等
· 赛力斯 86.40（-3.01%）— 跟随新能源板块回调，趋势没坏，不动

总浮亏 -15.2%，今天没有需要动手的。

自选关注：xxx 放量突破 MA20，明天盯一下开盘。
---
```

### 触发方式

- 用 OpenClaw cron，每个交易日收盘后自动触发（北京时间 17:00）
- **交易日检测**：调 `scan_market` 后检查返回的 `timestamp`——如果日期不是今天，说明今天没开盘，跳过不发。这样零维护，不需要额外维护节假日表
- 手动触发：微信里说"今日复盘"/"盘后报告"/"看看盘"/"今天咋样"时立即生成

## 技术要点

- TideWatch MCP 在 `localhost:8889`，你可以直接 HTTP 调用
- 核心工具：`scan_market`（批量扫描）、`analyze_stock`（单股详情）
- **不需要**单独调 `manage_holdings` / `manage_watchlist` 拿列表——`scan_market` 返回的 `holdings[]` 和 `watchlist[]` 已包含完整数据（名称、价格、涨跌、score、信号、成本、盈亏）
- 画图脚本建议放在 `~/.openclaw/workspace/scripts/tidewatch_chart.py`
- matplotlib 如果没装：`pip install matplotlib`

### analyze_stock 调用策略

`scan_market` 已经够写大部分报告了。**不要对每只持仓都调 `analyze_stock`**（每只 2-3 秒，全调太慢）。只在满足以下任一条件时才深入分析：

- 当日涨跌幅 > 3%（异动）
- score 绝对值 > 70（强信号）
- signal 字段非空（有新信号触发）
- `pct_today` 与持仓整体方向严重背离

其他持仓用 `scan_market` 返回的数据一笔带过（"其他几只继续躺着"）。

### 异常处理

- **scan_market 超时/失败**：发一条"今天 TideWatch 数据暂时拉不到，我稍后再试" → 30 分钟后自动重试一次
- **部分持仓数据缺失**（baostock 瞬断常有）：正常发报告，缺失的标注"数据暂缺"
- **完全无数据**：不发报告，避免发空内容
- **画图脚本崩溃**：降级为纯文字报告，不阻塞整个流程

## 周五特别版

周五的报告多加一段 **本周回顾**：

```
📅 本周回顾：持仓整体 +2.3%，信号命中率 65%（13/20）
最佳：中际旭创 +8.2%  最差：药明康德 -3.1%
```

数据来源：`review_signals(days=5)` 拿信号命中率，`scan_market` 各持仓的周涨跌可从 sparkline 或当周累计算出。
